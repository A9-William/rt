Requestor(s)

The Requestor is usually only the one person that initiated the Ticket, but more requestors can be added. The Requestor(s) will (dependent on the RT configuration) get correspondance sent to them. With the default RT configuration, RT should be transparent to the requestor(s). Be aware that things might go a bit wrong when multiple requestors and/or Ccs are used; persons replying to mail might typically send reply to all receipients - which is bad because RT might duplicate the reply to those persons. <& /Ticket/Elements/EditWatcherList, Watchers=>$Requestors, Type=>'Requestor', Scope=>'Ticket', id=>$id &> %# Edit Cc receipients
